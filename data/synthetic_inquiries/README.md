# Synthetic Inquiry Records

## Intake-only records

The `intake_only` records represent what may be available at the beginning of the workflow. They intentionally leave `review_summary` and `derived_assessment` as null.

## Finalized examples

The `finalized_examples` folder shows how the same carry-forward schema can be filled after reviewer investigation and confirmation.

These finalized examples are included for schema demonstration only. They are not meant to imply that reviewers should hand-edit YAML in real life. In a real workflow, a form, database record, or review-note interface would collect these fields.
